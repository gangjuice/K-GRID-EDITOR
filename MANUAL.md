## 프로그램 개요

**K-GRID EDITOR**는 네이버 지도와 GeoJSON 도면을 기반으로 필지 정보를 관리하고 입력할 수 있는 데스크톱 애플리케이션입니다.

### 주요 특징
- ✅ **이중 모드**: 사외망(인터넷)과 사내망(오프라인) 두 가지 환경 지원
- ✅ **GeoJSON 기반**: 토지이용계획 도면을 GeoJSON 형식으로 로드
- ✅ **필지별 관리**: 각 필지의 용도, 면적, 수요전력, 공급률 등 입력
- ✅ **데이터 동기화**: xlsx 파일로 데이터 내보내기/불러오기
- ✅ **자동 저장**: 로컬스토리지 + 로컬 디스크에 자동 저장
- ✅ **다양한 시각화**: 상태별, 그라데이션, 변전소별, 배전선로별 보기

## 시스템 요구사항

### 필수 사항
- **OS**: Windows 7 이상
- **RAM**: 최소 2GB (권장 4GB 이상)
- **저장공간**: 최소 500MB
- **Node.js**: 설치되어 있어야 함 (버전 14.0 이상)

### 선택 사항 (사외망 사용 시)
- **인터넷 연결**: 네이버 지도 표시를 위해 필요
- **웹 브라우저**: Chrome, Edge, Firefox 등 (Electron에 포함)

## 설치 및 실행

### 1단계: 프로그램 다운로드
1. K-GRID-EDITOR 폴더를 원하는 위치에 다운로드/복사

### 2단계: 첫 실행 (필수 - 한 번만)
1. **build.bat** 파일을 더블클릭
2. 자동으로 필요한 라이브러리 설치 및 exe 파일 생성
3. 완료 후 `dist` 폴더에 **K-GRID-EDITOR-1.0.0-portable.exe** 생성

### 3단계: 이후 실행
- **dist 폴더의 .exe 파일**을 더블클릭하여 실행
- 또는 build.bat를 다시 실행하여 최신 버전으로 빌드

### 개발 환경에서 실행 (개발자용)
```powershell
# PowerShell에서 실행
$env:PATH = "C:\Program Files\nodejs;" + $env:PATH
cd "K-GRID-EDITOR 폴더 경로"
npm start
```

### 상단 메뉴바

#### 🌐 사외망 / 🔒 사내망 버튼
- 왼쪽 상단에 위치
- **사외망**: 네이버 지도 + 인터넷 필요
- **사내망**: Leaflet + 완전 오프라인 작동
- 클릭하면 모드 전환

#### 📄 파일 메뉴
- **도면 레이아웃 열기 (.geojson)**: 토지이용계획 도면 파일 로드

- **데이터 내보내기 (.xlsx)**: 현재 작업 데이터 저장

#### 👁️ 보기 메뉴
- **상태별 기본 보기**: 필지 상태별로 색상 구분
  - 🔵 파란색: 송전완료
  - 🔴 빨간색: 미송전
  - ⚪ 하얀색: 무부하
  - 🟡 노란색: 미입력

- **공급률 그라데이션 보기**: 공급률에 따라 색상 그라데이션 표시
- **변전소별 필지 보기**: 같은 변전소의 필지를 같은 색상으로 표시
- **배전선로별 필지 보기**: 같은 배전선로의 필지를 같은 색상으로 표시
- **토지이용계획 도면 켜기/끄기**: 도면 표시 여부 선택
- **블럭명 오버레이 켜기/끄기**: 필지명 텍스트 표시 여부

#### 📊 데이터 관리 메뉴
- **필지 데이터 대시보드**: 모든 필지 정보를 테이블로 표시
- **미입력 필지 찾기**: 정보가 미입력된 필지 목록 표시
- **사내 결재 리포트**: (향후 기능)

#### ⚙️ 기타 메뉴
- **도움말**: 조작법 및 단축키
- **환경 설정**: 도면 투명도, 편집 보안 등
- **단위 면적당 수요전력 설정**: 용도별 기본 수요전력값 설정

### 좌측 마우스 패널 (선택 모드)
- **마우스 다중 선택: OFF/ON** 토글
  - OFF 상태: 필지 좌클릭 시 상세정보 편집
  - ON 상태: 필지 좌클릭 시 선택/해제, 일괄 변경 가능

### 우측 패널
- **범례**: 변전소별/배전선로별 색상 범례

## 사외망 vs 사내망

### 사외망 (index.html)
```
특징: 네이버 지도 + 인터넷
장점:
  • 실제 지도와 함께 필지 위치 확인 가능
  • 위성지도 보기 지원
  • 지도 배경 켜기/끄기 가능

단점:
  • 인터넷 연결 필수
  • 속도가 상대적으로 느림

사용 환경:
  • 사무실 (인터넷 있음)
  • 초기 설계 단계
```

### 사내망 (intranet.html)
```
특징: Leaflet 지도 + 완전 오프라인
장점:
  • 인터넷 없이도 100% 작동
  • 모든 라이브러리 로컬 저장
  • 빠른 성능

단점:
  • 배경 지도 없음 (회색 배면만)
  • 네이버 지도 기능 미지원

사용 환경:
  • 폐쇄 사내망
  • 현장 작업 (모바일 또는 오프라인)
  • 보안 이슈가 있는 환경
```

### 데이터 연동
- **사외망에서 작업 → xlsx 내보내기 → 사내망에서 같은 geojson + xlsx 로드 → 완벽 동기화**
- xlsx 파일 형식이 동일하므로 두 환경 간 자유로운 데이터 이동 가능

## 주요 기능

### 1. 도면 로드하기

#### 단계
1. **파일** → **도면 레이아웃 열기 (.geojson)** 클릭
2. 토지이용계획 도면 파일 선택 (geojson 형식)
3. 지도 위에 필지 구획 자동 표시

#### GeoJSON 파일 요구사항
```json
{
  "type": "FeatureCollection",
  "features": [
    {
      "type": "Feature",
      "properties": {
        "blockName": "필지명",
        "blockType": "용도",
        "area": 1234.56,
        "zone": "지역"
      },
      "geometry": {
        "type": "Polygon",
        "coordinates": [[[lng, lat], ...]]
      }
    }
  ]
}
```

---

### 2. 필지 정보 입력 (좌클릭)

#### 필지 클릭 시 나타나는 정보 창
```
[블럭명] 상세정보

필지 면적 (m²): ________
수요 예측부하 (kW): ________

필지 공급률: [슬라이더] 0-100%

공급완료 부하: 0.0 kW
미송전 부하: 0.0 kW

변전소명: ________ (선택/입력)
선로명 (D/L): ________ (선택/입력)
메모: ________________

[취소] [저장]
```

#### 입력 방법
1. 필지 클릭 → 상세정보 창 열림
2. 필지 면적 입력 (m²)
3. 수요 예측부하 입력 (kW) 또는 자동 계산 가능
4. 공급률 슬라이더로 조정 (0~100%)
5. 변전소명 및 선로명 입력/선택
6. 메모 입력 (선택사항)
7. **저장** 클릭

#### 자동 계산
- 공급완료 부하 = 수요 예측부하 × 공급률
- 미송전 부하 = 수요 예측부하 × (1 - 공급률)

---

### 3. 필지 상태 변경 (우클릭)

#### 상태 순환 변경
```
우클릭 시 상태가 자동 순환:

🔵 송전완료 
    ↓
🔴 미송전
    ↓
⚪ 무부하
    ↓
🟡 미입력
    ↓ (다시 송전완료로)
```

#### 자동 상태 설정
- **무부하 구간** (단위 부하 = 0인 용도): 자동으로 "무부하" 상태 설정
  - 예: 도로, 공원, 녹지, 광장 등

---

### 4. 일괄 변경 (마우스 다중 선택 모드)

#### 단계
1. **👆 마우스 다중 선택: ON** 클릭
   - 버튼이 🔴으로 변경되고 텍스트가 "ON"으로 표시
2. 변경할 필지들을 하나씩 좌클릭하여 선택
   - 선택된 필지는 빨간색 테두리 표시
3. **✏️ 선택 필지 일괄 변경** 버튼 클릭
4. 일괄 변경 창에서:
   - 변전소명 입력 (빈칸 시 유지)
   - 선로명 입력 (빈칸 시 유지)
   - 메모 입력 (빈칸 시 유지)
5. **일괄 적용** 클릭

#### 예시
- 50개의 필지를 "변전소A"로 일괄 설정 가능
- 일부 필지는 기존 데이터 유지, 일부는 수정

---

### 5. 대시보드 (필지 데이터 테이블)

#### 접근 방법
**데이터 관리** → **필지 데이터 대시보드**

#### 기능
| 기능 | 설명 |
|------|------|
| 정렬 | 각 열의 헤더를 클릭하여 오름/내림차순 정렬 |
| 필터링 | ID, 블럭명, 용도, 상태 등으로 필터링 |
| 다중 선택 | 체크박스로 여러 필지 선택 |
| 일괄 변경 | 선택한 필지들의 데이터 일괄 수정 |
| 지도 이동 | 행 더블클릭하면 지도가 해당 필지로 이동 |
| 합계 | 필터 기준 합계값 자동 계산 |

#### 열 설명
- **ID**: 자동 부여 번호
- **블럭명**: 필지 이름
- **용도**: 토지 이용 용도
- **필지 면적**: m² 단위
- **수요 예측(kW)**: 예상 전력 수요
- **공급률**: 0~100%
- **공급완료 부하**: 현재 공급 중인 부하
- **미송전부하**: 아직 공급하지 않은 부하
- **상태**: 송전완료/미송전/무부하/미입력
- **변전소명**: 담당 변전소
- **배전선로**: 연결된 배전선로
- **메모**: 추가 정보

---

### 6. 미입력 필지 찾기

#### 접근 방법
**데이터 관리** → **미입력 필지 찾기**

#### 기능
- 정보가 입력되지 않은 필지 목록 표시
- 각 필지 클릭 시 지도에서 해당 위치로 이동
- 빠르게 미입력 필지를 찾아 정보 입력 가능

---

### 7. 환경 설정

#### 접근 방법
**기타** → **환경 설정**

#### 설정 항목

**👁️ 도면 시각화 설정**
- **도면 투명도**: 0% (선명) ~ 100% (투명)
  - 낮을수록 도면이 선명하게 보임
  - 높을수록 지도가 더 잘 보임

**🔒 편집 보안 제어**
- **무부하 구간(0 kW/m²) 편집 잠금**
  - 체크: 수요전력이 0인 용도는 편집 불가
  - 체크 해제: 모든 필지 편집 가능

**⚠️ 위험 구역 (Danger Zone)**
- **현재 프로젝트 작업 데이터 전체 초기화**
  - 클릭하면 모든 입력 데이터 삭제 (복구 불가)

---

### 8. 수요전력 설정

#### 접근 방법
**기타** → **단위 면적당 수요전력 설정**

#### 기능
- 용도별 단위 수요전력값 설정 (kW/m²)
- 필지 면적 × 단위수요전력 = 예상 수요 자동 계산

#### 용도 카테고리
```
주거시설
  • 단독주택: 0.05 kW/m²
  • 아파트: 0.08 kW/m²
  • 도시형생활주택용지: 0.06 kW/m²
  • 주상복합: 0.1 kW/m²

상업·업무시설
  • 상업용지: 0.15 kW/m²
  • 근린생활시설용지: 0.1 kW/m²
  • 업무시설: 0.12 kW/m²
  ... (등)

산업·지원시설, 공공·교육시설, 교통시설, 녹지·공원, 유틸리티
```

#### 새 유형 추가
1. **➕ 유형 추가** 버튼 클릭
2. **팝업 창에서:**
   - 카테고리 선택 (주거시설, 상업·업무 등)
   - 용도명 입력 (예: "신규시설")
   - 단위 수요전력 입력 (예: 0.15)
3. **✓ 추가** 클릭
4. **저장** 클릭하여 적용

## 단계별 작업 가이드

### 시나리오 1: 새로운 프로젝트 시작

#### 1단계: 도면 로드
1. 프로그램 실행
2. **파일** → **도면 레이아웃 열기 (.geojson)**
3. 토지이용계획 도면 파일 선택
4. 지도에 필지 표시됨

#### 2단계: 필지 정보 입력
1. 입력하고 싶은 필지 좌클릭
2. 상세정보 창 열림
3. 다음 정보 입력:
   - 필지 면적 (이미 계산된 경우 확인)
   - 수요 예측부하 입력
   - 공급률 조정
   - 변전소명 선택/입력
   - 선로명 선택/입력
   - 메모 입력 (선택)
4. **저장** 클릭
5. 자동으로 다음 필지로 이동하거나 반복

#### 3단계: 데이터 저장
- 자동 저장 진행 중 (로컬스토리지 + 로컬 디스크)
- 작업 종료 시 **파일** → **데이터 내보내기 (.xlsx)**로 백업

### 시나리오 2: 이전 작업 계속하기

#### 2단계: 이전 데이터 복구
1. **파일** → **데이터 불러오기 (.xlsx)**

- 또는 기존 데이터 수정

### 시나리오 3: 사외망에서 사내망으로 이동

#### 1단계: 사외망에서 데이터 내보내기
1. 사외망 환경에서 작업 완료
2. **파일** → **데이터 내보내기 (.xlsx)**
3. 파일명: `프로젝트명_data.xlsx` (저장)

#### 2단계: 사내망 전환
1. 프로그램 상단 **🌐 사외망** 버튼 클릭
2. **🔒 사내망** 모드로 전환

#### 3단계: 도면과 데이터 로드
1. **파일** → **도면 레이아웃 열기 (.geojson)**
   - 같은 도면 파일 선택
2. **파일** → **데이터 불러오기 (.xlsx)**
   - 방금 내보낸 xlsx 파일 선택
3. 모든 데이터가 동일하게 복구됨

## 팁과 트러블슈팅

### 💡 팁

#### 효율적인 데이터 입력
- **공급률 슬라이더**: 마우스로 드래그하면 0~100% 빠르게 조정 가능
- **자동 계산**: 수요 예측부하 입력 시 공급완료/미송전 부하가 자동 계산됨
- **datalist 기능**: 변전소명, 선로명은 이전 입력값이 자동 제안됨

#### 데이터 백업
- 중요한 작업마다 **데이터 내보내기**로 xlsx 백업
- 로컬 디스크에 `data/` 폴더에 JSON으로도 자동 저장됨

#### 대시보드 활용
- 필터링으로 특정 용도의 필지만 보기
- 정렬로 수요전력 많은 순서대로 보기
- 합계로 전체 면적, 수요전력 확인

#### 시각화 모드 활용
- **공급률 그라데이션**: 공급률 분포를 시각적으로 파악
- **변전소별 보기**: 변전소별 부하 분포 확인
- **배전선로별 보기**: 선로별 부하 균형 확인

#### 성능 최적화
- 필지가 1,000개 이상인 경우 대시보드는 필터 사용 권장
- 도면 투명도를 50% 정도로 설정하면 가독성 좋음

### 🔧 트러블슈팅

#### Q1: 프로그램이 실행 안 됨
**A:** 
1. Node.js 설치 확인: `node -v` in PowerShell
2. build.bat 다시 실행
3. antivirus 소프트웨어 차단 여부 확인
4. C:\Users\[username]\AppData\Local\K-GRID-EDITOR 폴더 확인

#### Q2: 도면이 로드되지 않음
**A:**
1. GeoJSON 파일 형식 확인 (온라인 GeoJSON validator 사용)
2. 파일명에 한글 포함 시 UTF-8 인코딩 확인
3. coordinates 형식 확인: [lng, lat] 순서 (lat, lng 아님)

#### Q3: 필지 클릭해도 반응 없음
**A:**
1. **환경 설정** → "무부하 구간 편집 잠금" 확인
   - 체크된 경우: 0 kW/m² 용도는 수정 불가
   - 체크 해제하면 편집 가능
2. GeoJSON의 blockType 필드가 올바른지 확인
3. 다른 필지 클릭 시도 (특정 필지만 문제인지 확인)

#### Q4: 데이터가 저장되지 않음
**A:**
1. **로컬 디스크 자동 저장** 확인
   - `K-GRID-EDITOR/data/프로젝트명.json` 파일 생성 확인
   - 파일 권한 확인 (읽기/쓰기 권한 필요)
2. localStorage 비활성화 확인
   - F12 → Application → Cookies → localStorage 확인
3. 프로그램 종료 후 재실행

#### Q5: 사외망/사내망 전환 안 됨
**A:**
1. 상단 버튼 다시 클릭 (약간의 딜레이 있을 수 있음)
2. F5로 브라우저 새로고침
3. 프로그램 재실행

#### Q6: xlsx 불러오기/내보내기 실패
**A:**
1. xlsx 파일이 올바른 프로젝트의 데이터인지 확인
2. 파일명에 특수문자 없는지 확인
3. 다른 프로그램에서 xlsx 파일이 열려있지 않은지 확인

#### Q7: 사내망에서 느림
**A:**
1. 필지 수가 많은 경우 일반적인 현상 (로컬 처리)
2. 불필요한 오버레이(블럭명 등) 끄기
3. 도면 투명도를 높여서 렌더링 부하 감소

#### Q8: 데이터를 잘못 삭제함
**A:**
1. **환경 설정** → "데이터 전체 초기화" 클릭 이전이면 Ctrl+Z 시도
2. 이전에 내보낸 xlsx 파일이 있으면 **데이터 불러오기**로 복구
3. `K-GRID-EDITOR/data/` 폴더의 JSON 파일 확인 (자동 저장분)
